# Contributing

Thanks for your interest in improving this STM32 PZEM-004T project.

## How to Contribute

1. Fork the repository.
2. Create a feature branch.
3. Make focused changes.
4. Test on hardware when possible.
5. Open a pull request with clear details.

## Pull Request Checklist

- Keep changes small and scoped.
- Update documentation when behavior changes.
- Preserve coding style used in `Core/Src` and `Core/Inc`.
- Include wiring or hardware notes if required for testing.
- Mention MCU target and toolchain (for example STM32F103C8T6 + STM32CubeIDE).

## Bug Reports

Please include:

- Board and MCU (for example Blue Pill STM32F103C8T6)
- PZEM module version
- UART pins and baud rate
- Expected behavior vs actual behavior
- Serial debug output logs

## Feature Requests

Describe the use case and expected API behavior.
